# FDB-ProyectoBiblioteca
Nombre del proyecto: 
Proyecto Biblioteca.

Motivacion: 
Facilitar el control de prestamos y devoluciones en la biblioteca de la institucion.

Descripcion del pryecto:
El proyectro es un sistema de gestion para los prestamos y devoluciones de los libros en la bibliotica, donde se reflejaran los datos del estudiante, el libro que se va a prestar,
si es copia o no; la fecha y la hora del prestamo, asi como la fecha y la hora de la devolucion, si tiene retrado o alguna devolucion pendiente.

¿Que cambios y forma normal se utilizo al normalizar los diagramas de bases de datos?
En el caso del primer diagrama "Diagrama E-R Normalizado", se corrijio la entidad de libro, paso de ser plural a individual y se corrijieron las cardinalidades correspondientes, tambien se eliminaron algunos
atributos para hacerlo menos redundante, y finalmente se tomo un atributo y se transformo en entidad con su propio atributo, normalizando asi el diagrama. 

Para el segundo diagrama "Diagrama E-R UML Normalizado", para este diagrama se hicieron algunos cosas, primero, se cambiaron alguna de las claves tanto foraneas como primarias para hacer referencias a sus
tablas correspondientes, al igual que en el diagrama anterior se creo una tabla derivada de un atributo, normalizando asi el diagrama UML. La forma normal que se esta utilizando la tercera forma normal 
(3FN), esto en ambos diagramas.
